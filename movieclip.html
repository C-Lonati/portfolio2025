구현할 때 생긴 문제점

1. 로딩시간에 대한 고민(영상 로딩속도)
웹 통계 자료를 찾아보던 중 아마존에서 웹 로딩 0.1초 지연되면 매출 1%감소로 이어진다는 이야기를 접했다.
해당 페이지에서도 페이지의 마지막 부분에 iframe으로 유튜브 영상을 불러오기 때문에 페이지 로딩 속도가 다른 프로젝트들에 비해 느린 편이었다.
비동기 로딩과 유사한 효과를 낼 수 있는 방법을 떠올려 해당 방법을 사용했다.
if(window.scrollY > storyHeight)
해당 if문으로 스크롤이 일정량 내려온 것을 감지 한 뒤 innerHTML을 통해 iframe문을 불러왔다.
if (!loading){ soundTrack.innerHTML = ''; loading = true; }
2. 패럴렉스 구현에 대한 고민
라이브러리를 사용하면 js에 비해 로딩 속도가 느려지기에 gsap을 이용해 구현할지, 라이브러리 없이 구현할지에 대해 고민했다.
포트폴리오에서 gsap을 사용하기로 결정했기에 별도의 라이브러리 없이 구현하기로 결정했다.
구현 결과 gsap을 사용하는쪽이 유지보수가 훨씬 쉽다는 결론에 도달했다.
3. iframe 프리징 문제
1에서 구현한 코드로 인해 윈도우 프리징 발생, iframe과다 생성이 원인으로 되었다.
기존 조건문 검사-innerhtml수정-조건문 종료 후 변수 수정되던 코드를 조건문검사-변수 수정-innerhtml수정 후 조건문 종료 순으로 변경
if (!loading){ loading = true; soundTrack.innerHTML = ''; }
